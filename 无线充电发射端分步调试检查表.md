# 无线充电发射端分步调试检查表

适用结构：

```text
STM32F103
→ 半桥驱动芯片
→ MOSFET 全桥逆变
→ LCC 谐振补偿
→ 发射线圈
```

调试总原则：

```text
先看小信号，再看驱动信号；
先低压空载，再低压带功率级；
先确认不会直通，再接谐振网络；
任何一步异常，都不要进入下一步。
```

## 第 1 步：只看 STM32 输出波形

此时不要接 MOSFET，不要接母线电压。

只让 STM32 工作，用示波器看 MCU 输出脚。

默认检查引脚：

```text
PA8   TIM1_CH1   → 半桥 A 高边输入信号
PB13  TIM1_CH1N  → 半桥 A 低边输入信号

PA9   TIM1_CH2   → 半桥 B 高边输入信号
PB14  TIM1_CH2N  → 半桥 B 低边输入信号
```

## 1.1 正常现象

应该看到：

```text
PA8  有方波
PB13 有方波
PA9  有方波
PB14 有方波
```

同一个半桥内应该互补：

```text
PA8 高时，PB13 低
PA8 低时，PB13 高

PA9 高时，PB14 低
PA9 低时，PB14 高
```

两个桥臂应该反相：

```text
PA8 和 PA9 大约相差 180 度
```

频率应该接近代码设置：

```c
#define WPT_FREQ_START_HZ
#define WPT_FREQ_MAX_HZ
#define WPT_FREQ_MIN_HZ
```

电平应该是：

```text
低电平接近 0V
高电平接近 3.3V
```

## 1.2 异常怎么看

### 异常 A：四路都没有波形

表现：

```text
PA8、PB13、PA9、PB14 都是直流低电平或直流高电平
```

优先排查：

```text
1. 程序有没有真正烧录进去。
2. MCU 是否在运行，晶振是否起振。
3. SystemClock_Config() 是否卡死。
4. main() 是否进入 while(1)。
5. InverterStart() 有没有被执行。
6. TIM1 时钟有没有打开。
7. GPIO 是否配置成 GPIO_MODE_AF_PP。
```

代码重点看：

```c
SystemClock_Config();
MX_GPIO_Init();
MX_TIM1_Init();
InverterStart();
```

如果你的板子没有外部 8MHz 晶振，而代码使用 HSE，程序可能卡在时钟初始化。

这时要么改成 HSI 时钟，要么确认外部晶振焊接正常。

### 异常 B：只有 PA8/PA9 有波形，PB13/PB14 没有

表现：

```text
主通道有 PWM
互补通道没有 PWM
```

优先排查：

```text
1. PB13、PB14 是否被别的功能占用。
2. TIM1 的 CH1N、CH2N 是否真的打开。
3. TIM1 的 MOE 主输出是否打开。
4. GPIOB 时钟是否打开。
5. PB13、PB14 是否配置成复用推挽输出。
```

代码重点看：

```c
__HAL_TIM_MOE_ENABLE(&htim1);
TIM_CCER_CC1NE;
TIM_CCER_CC2NE;
GPIO_MODE_AF_PP;
```

### 异常 C：同一半桥上下管信号同时为高

表现：

```text
PA8 和 PB13 有重叠高电平
或者
PA9 和 PB14 有重叠高电平
```

这是严重异常。

不要接驱动芯片。

不要接 MOSFET。

优先排查：

```text
1. TIM1 互补输出极性是否配置错。
2. 驱动芯片输入是否有反相。
3. WPT_HIGH_SIDE_ACTIVE_HIGH 是否正确。
4. WPT_LOW_SIDE_ACTIVE_HIGH 是否正确。
5. 死区时间是否太小。
```

代码重点看：

```c
#define WPT_HIGH_SIDE_ACTIVE_HIGH
#define WPT_LOW_SIDE_ACTIVE_HIGH
#define WPT_DEADTIME_NS
```

建议初始死区：

```c
#define WPT_DEADTIME_NS 800UL
```

### 异常 D：PA8 和 PA9 不是反相

表现：

```text
两个桥臂同时高、同时低，或者相位关系不稳定
```

优先排查：

```text
1. CH1 是否使用 PWM1。
2. CH2 是否使用 PWM2。
3. CH1、CH2 的 CCR 是否都是 ARR / 2。
4. 是否误改了 TIM1 配置。
```

代码重点看：

```c
sConfigOC.OCMode = TIM_OCMODE_PWM1;
HAL_TIM_PWM_ConfigChannel(... TIM_CHANNEL_1);

sConfigOC.OCMode = TIM_OCMODE_PWM2;
HAL_TIM_PWM_ConfigChannel(... TIM_CHANNEL_2);
```

### 异常 E：频率不对

表现：

```text
代码设置 180kHz，示波器显示明显不是 180kHz
```

优先排查：

```text
1. 系统时钟是不是 72MHz。
2. 外部晶振是否是 8MHz。
3. TIM1_GetClockHz() 是否按你的时钟树计算正确。
4. 是否使用了 CubeMX 生成的其他时钟配置覆盖。
```

代码重点看：

```c
SystemClock_Config();
TIM1_GetClockHz();
InverterSetFrequency();
```

## 第 1 步通过标准

只有同时满足下面条件，才进入第 2 步：

```text
四路 PWM 都有；
同一半桥上下互补；
两个桥臂反相；
死区清楚可见；
频率和代码设置一致；
高电平能达到驱动芯片输入要求。
```

## 第 2 步：接驱动芯片，看 HO/LO

此时可以连接：

```text
STM32 PWM 输出 → 半桥驱动芯片输入
```

但仍然不要接 MOSFET 高压母线。

驱动芯片需要先接好：

```text
VCC/VDD 驱动电源
GND
自举二极管
自举电容
必要的输入下拉电阻
必要的 EN/SD 使能电平
```

示波器检查：

```text
驱动芯片 LIN 输入
驱动芯片 HIN 输入
驱动芯片 LO 输出
驱动芯片 HO 输出
驱动芯片 VS/HS 半桥中点
```

## 2.1 正常现象

低边输出：

```text
LO 应该跟 LIN 对应
LO 幅值通常接近 0V 到驱动电源电压
比如 0V 到 10V、12V、15V
```

高边输出：

```text
HO 是相对 VS/HS 节点的高边栅极输出
不能只用普通地夹看 HO 对 GND 就下结论
应该看 HO - VS 的电压
```

正常高边栅极驱动应该是：

```text
Vgs_high = HO - VS
高电平约等于驱动电源电压
低电平接近 0V
```

## 2.2 异常怎么看

### 异常 A：LO 没有输出

表现：

```text
LIN 有输入，LO 没有方波
```

优先排查：

```text
1. 驱动芯片 VCC/VDD 是否正常。
2. 驱动芯片 GND 是否和 STM32 GND 共地。
3. EN/SD/INH 引脚是否处于使能状态。
4. STM32 的 3.3V 高电平是否能被驱动芯片识别。
5. 驱动芯片是否进入欠压保护。
```

重点判断：

```text
如果驱动芯片要求 5V 输入，而 STM32 只有 3.3V，可能识别不可靠。
```

解决方向：

```text
换支持 3.3V 输入的驱动；
或者加高速电平转换；
不要用很慢的光耦或普通三极管随便转换高频 PWM。
```

### 异常 B：HO 没有输出

表现：

```text
LO 正常，HO 不正常
```

优先排查：

```text
1. 自举二极管方向是否正确。
2. 自举电容是否接在 VB 和 VS 之间。
3. 自举电容容量是否合适。
4. 低边是否有足够导通时间给自举电容充电。
5. 驱动芯片是否需要半桥中点 VS 有开关动作后 HO 才正常。
```

常见原因：

```text
高边驱动依靠自举供电。
如果半桥中点不动，或者低边一直不导通，自举电容可能充不上电。
```

### 异常 C：HO 对地看起来很奇怪

表现：

```text
HO 对 GND 波形乱跳，幅值很高或看不懂
```

这不一定是错误。

因为高边驱动是浮地输出。

正确看法：

```text
用差分探头看 HO - VS；
或者双通道示波器分别看 HO 和 VS，再用 Math 做 HO - VS。
```

不要直接用普通探头地夹夹到高边浮动节点。

否则可能短路。

### 异常 D：驱动芯片发热

表现：

```text
还没接 MOSFET，驱动芯片就明显发热
```

优先排查：

```text
1. VCC 是否过高。
2. 输出脚是否短路。
3. HO/LO 是否误接到地或电源。
4. 驱动芯片型号和引脚定义是否看错。
5. EN/SD 逻辑是否接反。
```

## 第 2 步通过标准

只有同时满足下面条件，才进入第 3 步：

```text
驱动芯片输入 HIN/LIN 正常；
LO 输出幅值正常；
HO - VS 输出幅值正常；
上下管驱动没有重叠导通；
驱动芯片不异常发热；
EN/SD 使能逻辑正确。
```

## 第 3 步：接 MOSFET，低压母线测试

此时接上 MOSFET 全桥，但母线电压必须低。

建议：

```text
母线电压：先 5V 到 12V
电源限流：先设很小，例如 0.3A 到 1A
先不要接 LCC 和线圈
可以先用电阻负载或小功率假负载
```

检查点：

```text
MOSFET 栅极电阻
栅极下拉电阻
驱动芯片到 MOS 栅极走线
上下管是否接反
体二极管方向
母线电容
半桥中点波形
```

## 3.1 正常现象

半桥中点应该有方波：

```text
半桥 A 中点在 0V 和母线电压之间切换
半桥 B 中点在 0V 和母线电压之间切换
两个桥臂中点大约反相
```

全桥输出端应该看到：

```text
差分电压在 +Vbus 和 -Vbus 之间切换
```

电源电流应该比较小且可控。

MOSFET 不应该快速发热。

## 3.2 异常怎么看

### 异常 A：一上电限流电源立刻打限流

表现：

```text
母线电压被拉低
电源电流顶到限流值
MOSFET 或驱动芯片发热
```

这是严重异常。

立刻断电。

优先排查：

```text
1. 同一半桥上下 MOS 是否同时导通。
2. 高边和低边 MOS 是否装反。
3. MOSFET D/S/G 引脚是否接错。
4. 驱动芯片 HO/LO 是否接反。
5. 死区是否太小。
6. 驱动输入极性是否反了。
7. PCB 是否有焊锡短路。
```

重点看：

```text
低压都打限流，绝对不要升高母线电压。
```

### 异常 B：半桥中点不翻转

表现：

```text
半桥中点一直在 0V
或者一直在 Vbus
```

优先排查：

```text
1. 对应高边或低边 MOS 是否没驱动。
2. HO - VS 是否正常。
3. LO - GND 是否正常。
4. 自举电容是否充电。
5. MOSFET 是否焊错或损坏。
```

### 异常 C：半桥中点波形有很大尖峰

表现：

```text
开关瞬间有很高尖峰；
波形振铃严重；
示波器看到超过 MOSFET 耐压风险。
```

优先排查：

```text
1. 探头接地线是否太长。
2. 母线电容是否离 MOSFET 太远。
3. 栅极电阻是否太小，开关太快。
4. PCB 回路面积是否太大。
5. 是否需要 RC 吸收或 TVS。
```

先确认测量方法。

普通探头长地线会把尖峰看得很夸张。

建议用短地弹簧。

### 异常 D：MOSFET 很快发热

表现：

```text
低压、小电流下 MOSFET 也很快烫
```

优先排查：

```text
1. 栅极驱动电压是否太低，MOS 没完全导通。
2. 死区是否太小导致直通。
3. 死区是否过大导致体二极管长时间导通。
4. 开关频率是否太高。
5. MOSFET 型号是否适合高频开关。
6. 栅极电阻是否过大导致开关太慢。
```

## 第 3 步通过标准

只有同时满足下面条件，才进入第 4 步：

```text
低压母线不打限流；
两个半桥中点都正常翻转；
全桥差分输出正常；
MOSFET 不异常发热；
尖峰和振铃在可接受范围；
驱动电源稳定，没有掉压和欠压保护。
```

## 第 4 步：接 LCC 和发射线圈

这是最危险的一步。

因为 LCC 谐振后，电容和线圈上的电压、电流可能远大于母线电压和母线电流。

建议：

```text
仍然低压母线；
仍然限流；
先空载观察；
再放接收端；
逐步提高目标电流；
不要一开始就追求最大功率。
```

## 4.1 正常现象

低压扫频时应该看到：

```text
线圈电流随着频率变化；
靠近某些频率时电流明显增大；
接收端靠近时，发射端电流和波形会变化；
整流输出端能出现直流电压。
```

程序进入 RUN 后：

```text
线圈电流应围绕 WPT_TARGET_COIL_CURRENT_MA 附近变化；
频率会小步调整；
不会频繁故障停机。
```

## 4.2 异常怎么看

### 异常 A：一接 LCC 电流突然很大

表现：

```text
母线电源打限流；
线圈电流很大；
谐振电容发热；
MOSFET 发热；
程序进入故障。
```

立刻断电。

优先排查：

```text
1. LCC 电容值是否接错。
2. LCC 拓扑是否接错。
3. 线圈电感是否和设计差太多。
4. 当前频率是否正好打到强谐振点。
5. WPT_FREQ_MIN_HZ 是否太低。
6. WPT_TARGET_COIL_CURRENT_MA 是否设太高。
7. 扫频步长是否太大。
```

保守处理：

```c
#define WPT_TARGET_COIL_CURRENT_MA 500UL
#define WPT_SWEEP_CURRENT_MA       200UL
#define WPT_FREQ_MIN_HZ            120000UL
```

先把最低频率抬高，避免扫到危险谐振点。

### 异常 B：完全没有能量传过去

表现：

```text
接收端整流后没有电压；
发射端线圈电流很小；
频率怎么扫变化都不明显。
```

优先排查：

```text
1. 发射线圈是否断线。
2. LCC 元件是否接错。
3. 补偿电容容量是否偏太多。
4. 工作频率范围是否没有覆盖谐振点。
5. 接收线圈方向和距离是否合适。
6. 接收端串联谐振电容是否匹配。
7. 接收端整流桥方向是否接错。
```

重点：

```text
如果频率范围没覆盖谐振点，代码再调也没用。
```

### 异常 C：接收端电压很高但带不动负载

表现：

```text
空载电压很高；
一接负载电压就塌；
发射端电流不一定很大。
```

优先排查：

```text
1. 接收端整流二极管电流能力是否够。
2. 接收端滤波电容是否太小或 ESR 太高。
3. 接收端谐振电容是否不匹配。
4. 线圈耦合太弱，距离太远或偏移太大。
5. 发射端目标电流太低。
6. 发射端频率控制方向是否反了。
```

可能需要改：

```c
#define WPT_POWER_UP_BY_LOWER_FREQ 0
```

如果原来降频功率没变大，反而变小，就把这个宏反过来。

### 异常 D：程序频繁进入故障

表现：

```text
刚启动就停；
扫频一到某个点就停；
进入 RUN 后过几秒停。
```

优先排查：

```text
1. 是哪个保护触发。
2. ADC 采样比例是否正确。
3. 电流传感器零点是否正确。
4. raw 阈值是否设太低。
5. 谐振电压阈值是否设太低。
6. 温度 ADC 是否悬空。
```

重点宏：

```c
#define WPT_BUS_I_ZERO_MV
#define WPT_COIL_I_ZERO_MV
#define WPT_BUS_V_RAW_MAX
#define WPT_BUS_I_RAW_MAX
#define WPT_COIL_I_RAW_MAX
#define WPT_RES_V_RAW_MAX
#define WPT_TEMP_RAW_MAX
```

如果没接温度传感器：

```c
#define WPT_TEMP_RAW_MAX 4095UL
```

### 异常 E：谐振电容发热

表现：

```text
谐振电容很快发烫；
输出功率不大但电容温升明显。
```

优先排查：

```text
1. 电容类型是否适合高频大电流。
2. 电容 RMS 电流能力是否足够。
3. 是否用了普通电解或不合适的瓷片。
4. 工作频率是否让电容电流过大。
5. 是否需要多个薄膜电容并联分流。
```

谐振电容建议：

```text
优先用 CBB/薄膜电容；
耐压要留余量；
RMS 电流能力要留余量；
多个并联比单个硬扛更稳。
```

## 第 4 步通过标准

可以继续提高功率的条件：

```text
母线电流可控；
线圈电流可控；
谐振电压不过高；
MOSFET 不异常发热；
驱动芯片不异常发热；
谐振电容不异常发热；
接收端输出随距离和频率有合理变化；
保护阈值没有被频繁触发。
```

## 现场快速排查顺序

如果不知道哪里错了，按这个顺序查：

```text
1. STM32 四路 PWM 是否正确。
2. 驱动芯片 LO 是否正确。
3. 驱动芯片 HO - VS 是否正确。
4. MOSFET 半桥中点是否翻转。
5. 全桥差分输出是否正常。
6. LCC 接法是否正确。
7. 线圈电感和补偿电容是否匹配。
8. 接收端整流和滤波是否正确。
9. ADC 采样比例和保护阈值是否正确。
10. 频率控制方向是否正确。
```

## 绝对不要继续的红线

出现下面任何一种情况，都不要进入下一步：

```text
同一半桥上下驱动同时为高；
低压母线一上电就打限流；
MOSFET 低压下快速发热；
驱动芯片空载就发热；
HO - VS 驱动电压不正常；
半桥中点不翻转；
谐振电容快速发热；
示波器地线接法可能短路高边浮动节点。
```

## 建议的第一次安全参数

第一次接 LCC 和线圈时，可以先保守一点：

```c
#define WPT_FREQ_START_HZ          200000UL
#define WPT_FREQ_MIN_HZ            120000UL
#define WPT_FREQ_MAX_HZ            250000UL
#define WPT_FREQ_STEP_HZ              500UL

#define WPT_DEADTIME_NS               800UL

#define WPT_TARGET_COIL_CURRENT_MA    500UL
#define WPT_SWEEP_CURRENT_MA          200UL

#define WPT_BUS_OVERCURRENT_MA       1000UL
#define WPT_COIL_OVERCURRENT_MA      1500UL
```

确认稳定后，再逐步提高目标电流和母线电压。
